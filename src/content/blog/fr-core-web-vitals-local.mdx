## Qu'est-ce que les Core Web Vitals ?

Les Core Web Vitals (CWV) sont un ensemble de metriques definies par Google pour mesurer l'experience utilisateur sur un site web. Depuis 2021, elles font partie des criteres de classement de Google. Pour une entreprise locale dont le site est souvent le premier point de contact avec un prospect, ces metriques meritent une attention particuliere.

Les trois metriques principales sont :

- **LCP (Largest Contentful Paint)** : mesure le temps de chargement du contenu principal de la page. Google recommande un LCP inferieur a 2,5 secondes.
- **INP (Interaction to Next Paint)** : remplace le FID depuis 2024, mesure la reactivite de la page aux interactions utilisateur. Google recommande un INP inferieur a 200 millisecondes.
- **CLS (Cumulative Layout Shift)** : mesure la stabilite visuelle de la page. Google recommande un CLS inferieur a 0,1.

## Pourquoi c'est important pour le SEO local

Vous pourriez penser que les Core Web Vitals sont un sujet technique reserve aux grands sites. C'est une erreur. Voici pourquoi ils comptent pour votre activite locale.

### Impact sur le classement

Google utilise les CWV comme facteur de classement. A contenu egal entre deux sites concurrents, celui avec de meilleures performances techniques sera favorise. En SEO local, ou la concurrence se joue souvent entre quelques acteurs dans une zone geographique limitee, cet avantage peut faire la difference entre la premiere et la troisieme position.

### Experience mobile

Plus de 60 % des recherches locales se font sur mobile, souvent en situation de mobilite (dans la rue, en voiture). L'internaute a peu de patience : si votre page met plus de 3 secondes a charger, il retourne aux resultats et choisit un concurrent. Un site rapide et stable retient les visiteurs et augmente les conversions.

### Taux de rebond et signaux utilisateur

Un site lent genere un taux de rebond eleve (les visiteurs repartent rapidement). Google interprete ce comportement comme un signal negatif de qualite. A l'inverse, un site rapide ou les utilisateurs restent et interagissent envoie des signaux positifs.

## Comment mesurer vos Core Web Vitals

### Google PageSpeed Insights

L'outil gratuit de reference. Rendez-vous sur pagespeed.web.dev, entrez l'URL de votre site et obtenez un rapport detaille avec :

- Vos scores CWV reels (donnees de terrain, basees sur les visites reelles)
- Vos scores CWV de laboratoire (mesures dans des conditions controlees)
- Des recommandations d'amelioration classees par impact

### Google Search Console

Dans la section "Experience de la page", Search Console vous montre l'etat de vos CWV pour toutes les pages de votre site, classees en "Bon", "A ameliorer" et "Mediocre".

### Lighthouse (dans Chrome DevTools)

Pour les plus techniques, l'outil Lighthouse integre a Chrome permet une analyse detaillee page par page. Faites un clic droit > "Inspecter" > onglet "Lighthouse" > "Generate report".

### GTmetrix

Un outil complementaire qui offre une analyse approfondie avec des cascades de chargement visuelles, tres utile pour diagnostiquer les problemes specifiques.

## Optimisations rapides pour les sites de PME

Voici les actions les plus impactantes, classees par facilite de mise en oeuvre.

### Ameliorer le LCP (temps de chargement)

**Optimiser les images** : c'est le probleme numero un des sites de PME. Les photos prises au smartphone et envoyees directement sur le site pesent souvent plusieurs megaoctets.

- Convertissez vos images au format WebP (reduction de 25 a 35 % par rapport au JPEG)
- Redimensionnez-les a la taille d'affichage reelle (pas besoin d'une image de 4000 pixels si elle s'affiche en 800 pixels)
- Utilisez le chargement differe (lazy loading) pour les images sous la ligne de flottaison
- Compressez avec des outils comme TinyPNG, Squoosh ou ShortPixel

**Choisir un hebergement performant** : un hebergement mutualise a 3 euros par mois sera toujours plus lent qu'un hebergement de qualite. Pour un site local, des hebergeurs comme o2switch, Infomaniak ou OVH en offre Performance offrent un bon rapport qualite-prix.

**Utiliser un CDN** : un Content Delivery Network distribue vos fichiers depuis des serveurs proches de vos visiteurs. Cloudflare propose un CDN gratuit qui ameliore significativement les temps de chargement.

**Minimiser les ressources bloquantes** : reduisez le nombre de fichiers CSS et JavaScript qui bloquent le rendu de la page. Si vous utilisez WordPress, desactivez les plugins inutiles qui ajoutent leurs propres scripts.

### Ameliorer l'INP (reactivite)

- **Reduisez le JavaScript** : chaque script ajoute du temps de traitement. Supprimez les fonctionnalites non essentielles.
- **Evitez les carrousels complexes** : les sliders avec animations lourdes degradent l'INP. Une image statique bien choisie est souvent plus efficace.
- **Optimisez les scripts tiers** : les widgets de chat, les pixels de suivi et les polices externes ralentissent la reactivite. Chargez-les en mode asynchrone ou differe.

### Ameliorer le CLS (stabilite visuelle)

- **Definissez les dimensions des images** : ajoutez toujours les attributs width et height a vos balises img pour que le navigateur reserve l'espace avant le chargement.
- **Reservez l'espace des publicites** : si vous affichez des bannières, definissez une taille fixe pour l'emplacement.
- **Chargez les polices correctement** : utilisez font-display: swap dans vos declarations CSS pour eviter les sauts de texte au chargement des polices personnalisees.
- **Evitez l'insertion dynamique de contenu** : les bannieres de cookies, les barres de notification et les pop-ups qui poussent le contenu vers le bas degradent le CLS. Affichez-les en overlay plutot qu'en insertion.

## Optimisations specifiques a WordPress

WordPress etant le CMS le plus utilise par les PME, voici des recommandations specifiques.

- **Utilisez un plugin de cache** : WP Rocket (payant mais excellent), W3 Total Cache ou LiteSpeed Cache ameliorent considerablement les performances.
- **Limitez les plugins** : chaque plugin ajoute du code. Faites un audit regulier et supprimez ceux que vous n'utilisez pas. Visez moins de 15 plugins actifs.
- **Choisissez un theme leger** : les themes "tout-en-un" avec des dizaines de fonctionnalites sont souvent lents. Preferez un theme optimise comme GeneratePress, Astra ou Kadence.
- **Utilisez un plugin d'optimisation d'images** : Imagify, ShortPixel ou Smush compressent automatiquement vos images a l'upload.
- **Mettez a jour regulierement** : WordPress, les themes et les plugins. Les mises a jour incluent souvent des ameliorations de performance.

## Prioriser vos actions

Vous n'avez pas besoin de tout optimiser en une fois. Voici un ordre de priorite pragmatique.

1. **Optimisez les images** : c'est l'action avec le meilleur ratio effort/impact pour la plupart des sites de PME.
2. **Installez un plugin de cache** (si WordPress) : amelioration immediate et significative.
3. **Supprimez les plugins et scripts inutiles** : chaque element en moins accelere le site.
4. **Verifiez votre hebergement** : si votre site reste lent malgre les optimisations, le probleme vient peut-etre de la.
5. **Corrigez les problemes CLS** : ajoutez les dimensions aux images et optimisez le chargement des polices.

## Conclusion

Les Core Web Vitals ne sont pas qu'une preoccupation technique : ils ont un impact direct sur votre visibilite locale et sur la conversion de vos visiteurs en clients. La bonne nouvelle, c'est que les optimisations les plus impactantes (compression d'images, cache, suppression de scripts inutiles) sont accessibles meme sans competences techniques avancees. Commencez par mesurer vos performances actuelles avec PageSpeed Insights, identifiez les points faibles et traitez-les par ordre de priorite. Un site rapide et stable est un avantage concurrentiel reel en SEO local.
